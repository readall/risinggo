# RisingWave MCP Server Implementation Plan

## Overview

This plan outlines the phased implementation of the RisingWave MCP Server (Go implementation) based on the beads defined in `tasks/` and the specifications in `specs/`.

**Key Goals**:
- Strictly read-only MCP server with zero mutation capability
- p99 latency < 20ms under 200 concurrent sessions
- Production-grade observability and safety

## MCP Server Architecture

```
cmd/server/
├── main.go           # Entry point, transport selection

internal/
├── config/
│   └── config.go     # Env var loading, validation
├── db/
│   └── pool.go       # pgxpool management
├── safety/
│   ├── validator.go  # Query validation logic
│   └── keywords.go   # Mutation keyword blocklist
├── mcp/
│   ├── server.go     # MCP server bootstrap
│   └── toolregistry.go # Tool registration
├── tools/
│   ├── query.go      # execute_safe_read_query
│   ├── schema.go     # show_tables, describe_table
│   ├── explain.go    # explain tools
│   ├── monitoring.go # list_streaming_jobs
│   └── storage.go    # hummock_stats
├── observability/
│   ├── logging.go
│   ├── metrics.go
│   └── tracing.go
└── middleware/
    └── safety.go     # Validation middleware

tests/
├── unit/             # Unit tests for each component
└── integration/      # testcontainers-based tests
```
